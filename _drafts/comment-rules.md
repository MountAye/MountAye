引用编随的一篇文章：[《政治常识扫盲：澄清“言论自由”的各种误区》](https://program-think.blogspot.com/2014/02/freedom-of-speech.html)，这个链接在国内无法打开，稍后我将进行转载，并提供转载链接。

## 留言的处理办法

我一般只在周末打理博客，也就是周五下班后到周一上班前，具体时间不定。

每次有博文更新，我将在 Issues 新建一个和文章标题相同的 Milestone，与文章有关的评论都应该放在该 Milestone 之下。对于博客的通用和一般性建议，放在 General Comments 之下。编辑评论时，欢迎添加可在 Open Issues 中使用的标签（使用方法见下一节）。作为 Issue owner，如果想结束相关讨论，可以自行 close。

如果评论区的留言过多（希望能有那么一天），我将在一条 Issue 的最后一条回复之后一周的周末 close issue。（举个例子，1月23日周三的留言，“之后一周的周末”指的是2月1日至3日那个周末。）对于符合描述的 issue，将会添加对应的标签，必要时还会锁定会话（五毛和白学家一个也别想跑）。

对于没有锁定的 closed issues，本博客并不禁止挖坟。

本博客尽可能保障 issue owner 的言论自由，如果 issue owner 认为自己评论下的回复超出了自己可以忍受的范围，可以要求博主关闭、固定并添加负面标签，或者直接要求删除。请求将根据本博客的言论自由规则，立即处理（没有一周的缓冲期）。

## 标签：配合 close 和 lock conversation 功能

目前Issues区启用了如下几个标签：`正在讨论`、`集思广益`、`博客勘误`、`建议反馈`、`发言规则`、`博主推荐`、`论据失实`、`逻辑错误`、`有失风度`、`不知所云`。这些标签大致分为以下两类。

- 可以在 Open Issues 里使用的标签：

    + ### `正在讨论`

    + ### `集思广益`
    
    + ### `博客勘误`
    
    + ### `建议反馈`

    + ### `发言规则`

- 只在 Closed Issues 里使用的标签：

    + ### `博主推荐`

    + ### `论据失实`

    + ### `逻辑错误`

    + ### `不知所云`

        - 留言中的错别字、标点符号的不当使用、语病已经影响到了一般人的理解。

        - 单纯的观点表达，未对自己的观点提供必要的论证，或论证中存在逻辑错误，即有可能被标记为`逻辑错误`的留言（反之则不一定，标记为`逻辑错误`的留言在关闭之前不一定会标记为`不知所云`）。

        - 并非常识的，未提供有效出处（超链接、便于检索的关键字、书名，若被引出处较长，应提供具体位置并自行简要概括）的事实陈述，即有可能被标记为`论据失实`的留言（反之不一定）。

        - 垃圾信息

    + ### `有失风度`
