留言和评论

对于博客的一般性问题进行留言，请在 General Comments 区提交 issue。

评论博文，请点击博文结尾处提供的链接，或在 issue 区的 milestones 中寻找和博文同名的条目，新建或评论他人的 issues。

留言方法

本博客是一个静态网站，没有可以处理留言和评论的服务端。作为替代，本博客使用 GitHub 仓库的 issue 区作为评论区。每个 milestone 对应于一篇博文，每条 issue 对应于一层评论，每个 comment 对应于一条回复。

又由于 GitHub 的设定，除我以外的用户没办法指定 issue 对应的 milestone，所以评论方法有点复杂：

  • 根据在想要留言的页面上提供的链接找到 GitHub 仓库的 issue 区：
  • 在跳转的页面里，找到要评论的文章对应的 Milestone 编号，也就是图中黑色标题的中括号里的数字【34】：
  • 点击右上角的 create issue 按钮，再点击接下来页面的 get started 按钮，进入编辑区：
  • 在你评论的标题之前，输入带有黑色全角中括号的 Milestone 编号：
  • 阅读正文部分的提示后，删除提示内容，编辑留言内容，满意之后点击 Submit new issue 按钮。

留言规则

鉴于目前评论数量极少，尚未出现有必要管制的言论，暂时没有对成文法的强烈需求，所以暂时采用习惯法，依靠博主和读者的自觉来维持评论区秩序。

所有 issues 和 comments 需遵守 GitHub 社区相关规范：

  1. Term of Service: https://help.github.com/en/articles/github-terms-of-service
  2. Community Guidelines: https://help.github.com/en/articles/github-community-guidelines
